# SecretChat

基于非对称加密的 Flutter 安全聊天应用

## 功能特性

### 已实现 ✅

- **主密码保护**
  - 首次启动设置主密码
  - 使用 PBKDF2 密钥派生 (100,000 次迭代)
  - 应用启动期间保持解锁状态

- **非对称加密**
  - RSA-2048 密钥对生成
  - PEM 格式密钥导入/导出
  - AES-256-GCM 混合加密
  - RSA-OAEP 密钥封装

- **消息完整性**
  - HMAC-SHA256 完整性验证
  - RSA 数字签名
  - 防篡改检测

- **联系人管理**
  - 支持多个联系人
  - 为每个联系人独立配置公钥
  - 支持导入他人公钥

- **消息手动传输**
  - 导出加密消息（Base64 编码）
  - 导入加密消息并解密
  - 支持文本消息

- **Material UI**
  - 现代化浅色主题
  - 类似微信/WhatsApp 的布局
  - 对话列表界面
  - 聊天详情界面
  - 密钥管理界面

## 技术架构

### 加密流程

```
发送消息:
明文 → AES-256-GCM → 密文
AES 密钥 → RSA-OAEP(对方公钥) → 加密密钥
密文 → HMAC-SHA256 → 完整性标签
密文 → RSA 签名 (自己私钥) → 签名

接收消息:
验证 HMAC → 验证签名 → RSA 解密 AES 密钥 → AES 解密 → 明文
```

### 存储结构

```
应用文档目录/
└── secretchat/
    ├── salt.dat              # 随机盐
    ├── hash.dat              # 密码哈希
    └── keys/
        ├── {keyId}.pem       # RSA 私钥 (PEM 格式)
        └── contacts/
            └── {contactId}_pub.pem  # 联系人公钥
```

### 核心依赖

- `pointycastle` - 密码学算法
- `encrypt` - 高级加密封装
- `crypto` - HMAC/哈希
- `provider` - 状态管理
- `share_plus` - 消息导出分享

## 快速开始

### 环境要求

- Flutter SDK >= 3.8.0
- Dart >= 3.0.0

### 安装依赖

```bash
flutter pub get
```

### 运行应用

```bash
flutter run
```

### 使用流程

1. **首次启动**
   - 设置主密码（至少 8 位）
   - 自动生成 RSA-2048 密钥对

2. **日常使用**
   - 打开应用直接进入主界面
   - 无需每次输入密码

3. **添加联系人**
   - 点击右上角 + 添加联系人
   - 导入联系人的公钥（PEM 格式）

4. **发送消息**
   - 选择联系人进入聊天
   - 输入消息并点击发送
   - **输入主密码验证**（首次使用密钥时）
   - 自动加密并生成 Base64 字符串
   - 复制发送给对方

5. **接收消息**
   - 选择对应联系人
   - 点击导入按钮粘贴加密消息
   - **输入主密码验证**（解密消息）
   - 自动解密并验证

6. **安全锁定**
   - 设置 > 锁定密钥
   - 锁定后所有密钥操作都需要重新验证

## 安全性说明

### 当前实现

- ✅ 主密码使用 PBKDF2 派生密钥
- ✅ 主密码仅在密钥操作时验证
- ✅ 私钥以 PEM 格式存储（未加密）
- ✅ 消息双重验证（HMAC + 签名）
- ✅ 应用关闭后清除内存中的密钥
- ✅ 支持手动锁定密钥

### 安全建议

⚠️ **注意**: 当前版本私钥以 PEM 明文存储在生产环境中应：
1. 使用主密码加密私钥文件
2. 使用系统级安全存储（Keychain/Keystore）
3. 添加生物识别解锁

### 已知限制

- 私钥文件未加密存储（开发版本）
- 无自动备份功能
- 无密钥恢复机制
