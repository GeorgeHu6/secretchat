# 更新日志

## v0.1.1 - 改进认证流程

### 主要变更

#### ✅ 认证流程优化
- **不再每次启动都要求输入密码**
- 仅在以下情况需要验证主密码：
  - 生成新密钥对
  - 导入密钥
  - 导出私钥
  - 发送加密消息
  - 解密接收的消息
  - 手动锁定密钥后使用

#### 📝 新增功能
- **密码验证对话框** (`PasswordVerifyDialog`)
  - 统一的密码验证界面
  - 显示操作类型和警告信息
  - 密码错误时显示提示
  
- **密钥锁定功能**
  - 设置中添加"锁定密钥"按钮
  - 锁定后所有密钥操作都需要重新验证密码
  - 增强安全性

- **自动解锁**
  - 应用启动时自动进入主界面
  - 无需输入密码即可浏览联系人和设置
  - 仅在敏感操作时验证

### 使用流程变化

#### 旧版本 (v0.1.0)
```
启动应用 → 输入密码 → 进入主页 → 使用所有功能
```

#### 新版本 (v0.1.1)
```
启动应用 → 自动进入主页 → 浏览功能
     ↓
需要密钥操作 → 输入密码验证 → 继续操作
```

### 示例场景

#### 场景 1: 日常使用
1. 打开应用 → 直接进入主界面 ✅
2. 查看联系人列表 ✅
3. 浏览历史消息 ✅
4. 点击"发送消息" → 🔒 要求输入密码
5. 验证通过 → 发送加密消息

#### 场景 2: 生成密钥
1. 设置 → 密钥管理
2. 点击"生成密钥" → 🔒 要求输入密码
3. 验证通过 → 确认生成
4. 新密钥创建成功

#### 场景 3: 导出公钥
1. 密钥管理 → 点击密钥菜单
2. 选择"导出公钥" → ✅ 不需要密码（公钥可公开）
3. 分享给联系人

#### 场景 4: 导出私钥
1. 密钥管理 → 点击密钥菜单
2. 选择"导出私钥" → 🔒 要求输入密码
3. ⚠️ 显示安全警告
4. 确认后导出

### 安全性说明

#### 保护的内容
- ✅ 私钥加密操作（签名、解密）
